# UDP 프로토콜
## UDP가 하는 일
- 전송 방식은 너무 단순에서 신뢰성이 낮다. (순서가 바뀌거나 중복, 누락 등)
- 일반적으로 오류의 검사와 수정이 필요 없는 프로그램에서 수행할 것으로 가정

## UDP 프로토콜의 구조
- 총 8 Byte
- 출발지 포트번호 2 Byte
- 목적지 포트번호 2 Byte
- 길이 (페이로드 포함) 2 Byte
- Checksum (프로토콜 체크) 2 Byte

## UDP 프로토콜을 사용하는 프로그램
- 도메인을 물으면 IP를 알려주는 DNS 서버
- UDP로 파일을 공유하는 tftp 서버
- 라우팅 정보를 공유하는 RIP 프로토콜
